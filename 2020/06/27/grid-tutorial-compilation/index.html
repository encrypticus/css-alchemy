Исчерпывающее руководство по созданию модульной библиотеки, реализующей адаптивную сетку. Часть 8. Компиляция.

В предыдущей части мы реализовали последний модуль нашей библиотеки. В заключительной части мы сосредоточимся на компиляции исходных sass-файлов в файл css и поговорим о его оптимизации. На данный момент мы имеем готовую, полностью законченную библиотеку, реализующую адаптивную сетку. Полный исходный код проекта можно посмотреть на github. Также имеется подробная документация на русском и английском языках. Для компиляции исходников я буду использовать таск-менеджер Gulp.

Для начала установим утилиту командной строки Gulp:

npm i --global gulp-cli

В корневой директории библиотеки grid/ создадим файл package.json:

npm init

Это позволит задать имя, описание, версию проекта и другую нужную информацию.

Теперь установим Gulp в наши зависимости:

npm i --save-dev gulp

Также нам потребуются некоторые дополнительные пакеты.

Для компиляции sass-файлов нам понадобится плагин gulp-sass:

npm i --save-dev gulp-sass

Для оптимизации скомпилированного css-файла библиотеки на понадобится css-минификатор csso. Его мы будет использовать для сборки файла grid.min.css:

npm i --save-dev gulp-csso

При компиляции в конечном css-файле получится очень много одинаковых повторяющихся медиазапросов с разным кодом внутри:

@media screen and (max-width:1280px) {
  .justify-content-desktop-start {
    justify-content: flex-start
  }
}

@media screen and (max-width:1280px) {
  .justify-content-desktop-center {
    justify-content: center
  }
}

@media screen and (max-width:1280px) {
  .order-desktop-0 {
    order: 0
  }
}

Очевидно, что это неэффективно, так как намного увеличит размер css-файла. Для того, чтобы результат был таким:

@media screen and (max-width:1280px) {
  .justify-content-desktop-start {
    justify-content: flex-start
  }
  .justify-content-desktop-center {
    justify-content: center
  }
  .order-desktop-0 {
    order: 0
  }
}

мы должны установить плагин gulp-group-css-media-queries:

npm i --save-dev gulp-group-css-media-queries

У нас будут две версии css-файла – неминифицированная grid.css и минифицированная grid.min.css. Поэтому нужно установить плагин, позволяющий переименовывать файлы – gulp-rename:

npm i --save-dev gulp-rename

Также добавим плагин gulp-clean-css, который, как и csso является оптимизатором. Его мы будем использовать для сборки файла grid.css:

npm i --save-dev gulp-clean-css

Теперь в корень проекта добавим файл gulpfile.js и подключим все необходимы модули:

const gulp = require('gulp');
const sass = require('gulp-sass');
const csso = require('gulp-csso');
const groupCss = require('gulp-group-css-media-queries');
const rename = require('gulp-rename');
const cleanCss = require('gulp-clean-css');

Также добавим две задачи:

gulp.task('development', () => {
  return gulp.src('scss/grid.scss')
    .pipe(sass().on('error', sass.logError))
    .pipe(groupCss())
    .pipe(cleanCss({ format: 'beautify' }))
    .pipe(gulp.dest('dist'));
});

gulp.task('production', () => {
  return gulp.src('scss/grid.scss')
    .pipe(sass().on('error', sass.logError))
    .pipe(groupCss())
    .pipe(csso())
    .pipe(rename('grid.min.css'))
    .pipe(gulp.dest('dist'));
});

Задача development компилирует наш главный файл библиотеки grid.scss в файл grid.css и кладёт его в директорию dist/. Перед окончательной компиляцией файл проходит через группировку медиазапросов и оптимизацию с сохранением форматирования.

Задача production собирает файл grid.min.css также в директории dist, удаляя все пробелы и переводы строк.

Далее добавим задачу наблюдения за нашими scss-файлами, которая будет запускать пересборку при каждом изменении любого из файлов в директории scss и всех её поддиректорий:

gulp.task('watch', () => {
  gulp.watch(['scss/**/*.scss'], gulp.series(['development', 'production']));
});

И напоследок добавим дефолтную задачу, котороя последовательно запустит задачи development, production и watch:

gulp.task('default', gulp.series(
  'development',
  'production',
  gulp.series('watch')
));

Полный код файла gulpfile.js:

const gulp = require('gulp');
const sass = require('gulp-sass');
const csso = require('gulp-csso');
const groupCss = require('gulp-group-css-media-queries');
const rename = require('gulp-rename');
const cleanCss = require('gulp-clean-css');

gulp.task('development', () => {
  return gulp.src('grid/scss/grid.scss')
    .pipe(sass().on('error', sass.logError))
    .pipe(groupCss())
    .pipe(cleanCss({ format: 'beautify' }))
    .pipe(gulp.dest('dist'));
});

gulp.task('production', () => {
  return gulp.src('grid/scss/grid.scss')
    .pipe(sass().on('error', sass.logError))
    .pipe(groupCss())
    .pipe(csso())
    .pipe(rename('grid.min.css'))
    .pipe(gulp.dest('dist'));
});

gulp.task('watch', () => {
  gulp.watch(['scss/**/*.scss'], gulp.series(['development', 'production']));
});

gulp.task('default', gulp.series(
  'development',
  'production',
  'watch'
));

Теперь мы можем запустить компиляцию исхдников и наблюдение за файлами командой gulp в консоли в нашей корневой директории grid/.

Мы написали нашу библиотеку, используя scss-синтаксис препроцессора. Очевидно, что нам также нужна sass-версия библиотеки. Поэтому нам необходимо сконвертировать все наши scss-исходники в их sass-версии. Единственный адекватный способ конвертации scss в sass, что я нашёл – это утилита sass convert. Прикол в том, что эта утилита входит в состав дистрибутива Ruby. Поэтому для установки sass convert придётся немного поплясать с бубном. Для начала нужно установить пакет Ruby для вашей версии операционной системы. После установки у вас появится директория с установленным дистрибутивом Ruby. На моей Windows 10 это Ruby27-x64. Внутри директории Ruby27-x64 находится директория bin, в которой содержится нужный нам исполняемый файл утилиты sass-convert. Чтобы запускать утилиту из любого места системы, добавьте директорию bin в системную переменную PATH.

Теперь мы можем сконвертировать наши scss-файлы в sass. Перейдите в корневую директорию проекта grid/ и запустите следующую команду:

sass-convert -R scss --from scss --to sass

При указании опции -R утилита рекурсивно конвертирует все файлы в указанной директории и всех её поддиректориях. В нашем случае это директория scss. Опция --from задаёт формат, из которого производится конвертация, а --to – выходной формат.

После конвертации в директории scss и её поддиректориях mixins и partials рядом с scss-файлами появятся их sass-аналоги. Просто переместим их в директорию sass: файлы grid.sass и base.sass в корень директории, а миксины и части в их директории mixins и partials соответственно.
